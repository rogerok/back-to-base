# Types are just data

## `unknown` — the top of the subtyping hierarchy

unknown contains each and every type you will ever use in TypeScript.

**The union** of any type `A` with unknown will always evaluate to `unknow`, because, by defenition `A` is already contained in `unknown`

**The intersection** of any `A` type with unknown is the type `A`, because intersecting a set `A` with a set `B` means _extracting_ the part of `A` that also belongs to `B`.
Since any type `A` is inside `unknown`, `A` & unknown is just `A`.
