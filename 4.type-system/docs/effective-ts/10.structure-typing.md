# Структурная типизация

В приватных полях, структурная типизация тайпскрипта отключается и превращается в номинальную

```ts
class User {
  private type: string = "user";
}
class Admin {
  private type: string = "user";
}

const x: User = new User();
const b: Admin = x; // Error
```

```ts
const EMPTY_REF = Symbol("empty ref");

type MaybeEmptyRefCreator = {
  (): typeof EMPTY_REF;
  <T>(x: T): { value: T };
};

type TransparentWrapper = <A>(x: A) => A;

declare const refWrapper: MaybeEmptyRefCreator;

const wrap2 = (TransparentWrapper = (x) => x);

const wrap: TransparentWrapper = refWrapper; // Ошибка не возникает т.к. тайпскрипту достаточно найти одну перегрузку, чтобы считать тип корректным
```

При этом сигнатура ф-ции `type TransparentWrapper = <A>(x: A) => A;`, считается совместимой с `  (): typeof EMPTY_REF;`

Т.е. ф-ция сигнатура ф-ции, которая принимает один аргумент, совместима с ф-цией которая принимает ноль аргументов.

Это происходит из-за ковариативности - аргументы ф-ции могут быть шире, чем указанный тип, но не уже
