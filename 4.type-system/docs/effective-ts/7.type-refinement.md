# Основы вывода типов в Typescript. Как работает type refinement

Когда мы пишем код, и в нем есть переменная без указанного типа, TypeScript пытается вывести достаточно узкий тип,
при котором он будет удовлетворен

```ts
const a = 42; // type 42
let b = 42; // type number
```

Refinement в TypeScript - это процесс, когда TypeScript отрезает из union'a, те элементы, которые не могут случиться в
ветке выполнения -  
проверки typeof, instanceof, null/undefined, in, custom type-guard

Discriminated unions
---
Это наличие поля, которое позволяет нам понять что это за элемент union'a

```ts
type Event = { type: 'login', user: string } | { type: 'logout', timestamp: string }
```

Exhaustive Check
---

```ts
declare function asssertNever(x: never): void;

if (e.type === 'login') {
  console.log(e.user)
} else if (e.type === 'logout') {
  console.log(e.timestamp)
} else {
  assertNever(e);
}
```

}

Объектные типы позволяют иметь дополнительные поля
----
Поэтому возникнет ошибка т.к. никто не запрещает `ExternalUser` иметь поле `department`;

- `typeof` - безопасен
- уточнение по ПОЛЮ-дискриминатору - безопасно
- `instance` - безопасен*
- литералы - безопасны
- typeguard - безопасны*

Небезопасны
`in` - небезопасны
по не-дискриминаторам - небезопасны

```ts
type InternalUser = { name: string; department: string; subdepartment: string };
type ExternalUser = { name: string; company: string };
type User = InternalUser | ExternalUser;


function show(user: User) {
  if ('department' in user) {
    console.log(user.subdepartment.toUpperCase()) // user: IntertnalUser
  } else {
    console.log(user.company) // user: ExternalUser
  }
}

const obj = { name: 'John', company: 'Doe', department: 'S' };
const user: ExternalUser = obj;

show(user); // cannot read properties of undefined (reading 'toUpperCase');
```
