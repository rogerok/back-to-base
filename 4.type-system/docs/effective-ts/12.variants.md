```ts
interface Entity {
  id: string;
}

interface User extends Entity {
  name: string;
}

interface CompanyUser extends User {
  company: string;
}

type Processor1 = {
  name: string;
  handler(e: User): User;
};

const processor1: Processor1 = {
  name: "looger",
  handler(u: CompanyUser) {
    // Здесь нет ошибки, хоть и аргумент u является более узким типом
    console.log(u);
  },
};

type Processor2 = {
  name: string;
  handler: (e: User) => void;
};

const processor2: Processor = {
  name: "looger",
  handler: (u: CompanyUser) => {
    // Здесь ошибка возникает
    console.log(u);
  },
};
```

В объектах методы **бивариантны** - в объектах ф-циям мы можем передавать как более узкий тип, так и более широкий.
Это введено для совместимости.

Запись

```ts
type Processor1 = {
  name: string;
  handler(e: User): User;
};
```

Говорит о том, что `handler` является методом типа `Processor1` и он **бивариантен**.
**НО** бивариантность определяется только по входящим аргументам, поэтому такая запись спровоцирует ошибку

```ts
const processor1: Processor1 = {
  name: "looger",
  handler(u: CompanyUser) {
    console.log(u);
    return {} as CompanyUser;
  },
};
```
