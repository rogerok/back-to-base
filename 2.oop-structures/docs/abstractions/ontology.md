# Онтология

Описание объектов рассматриваемой области и связей между ними называется онтологией предметной области.
Связи можно описывать с помощью **ER-модели**

Иногда программисты думают наперёд и сразу формируют более сложные связи, например, m2m, а не o2m, что сказывается на сложности кода. Чем сложнее связь, тем больше кода и выше стоимость его создания и поддержки. Сложность связей можно описать так (правее — сложнее): o2o, o2m, m2m.

## Что такое предметная область (Domain)?

1. Предметная область — это та сфера реальной жизни, для которой создаётся программа. Каждая программа моделирует определённую часть реального мира:

Бухгалтерский софт → правила бухучёта, счета, проводки
Стриминговый сервис → сериалы, сезоны, эпизоды
Авиабилеты → рейсы, пассажиры, бронирования
E-commerce → товары, корзина, заказы

2. ## Сущности (Entities) — ключевые понятия

   Сущность — это объект предметной области, который имеет значение для бизнеса.

3. ## Связи между сущностями (Relationships)
   Сущности не существуют изолированно — они связаны друг с другом.

### Типы связей (по возрастанию сложности):

Один к одному (One-to-One, 1:1)
Урок ←→ Упражнение
Один урок имеет ровно одно упражнение
Одно упражнение принадлежит ровно одному уроку

Один ко многим (One-to-Many, 1:N)

Курс ←→ Уроки
Один курс содержит много уроков
Один урок принадлежит только одному курсу

Многие ко многим (Many-to-Many, M:N)
Пользователи ←→ Курсы
Один пользователь может проходить много курсов
Один курс могут проходить много пользователей
