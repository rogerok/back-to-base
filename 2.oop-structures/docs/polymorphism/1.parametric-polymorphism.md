# Параметрический полиморфизм

Проблема:

Допустим нам нужно написать функцию слияния двух массивов `concat`, в TypesScript функции, обрабатывающие массивы, ожидают определенный тип:

```ts
const concat = (arr1: number[], arr2: number[]): number[] => {
  ...
}
```

Для того чтобы не создавать на каждый тип массива новую функцию, нам пригодится параметрический полиморфизм.

Т.е. мы можем создать ф-цию-дженерик

```ts
const concat = <T>(arr1: T[], arr2: T[]): T[] => {
  ...
}
```

Параметрический полиморфизм дает возможность писать обобщенные алгоритмы для составных типов.
В динамических языках такой полиморфизм не нужен.
