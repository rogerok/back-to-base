# Конструктор

## Оператор new

В JavaScript есть встроенная поддержка генерации объектов.

```ts
// Такую функцию принято называть конструктором (хотя технически это обычная функция с контекстом)
// Конструкторы пишутся с заглавной буквы
function Company(name, website) {
  this.name = name
  this.website = website
  // Методы по-прежнему определены снаружи как обычные функции
  this.getName = getName
  this.getWebsite = getWebsite
}
```

Самое интересное в этом примере – оператор `new` (как и многое в js, он работает не так, как new в других языках).
Фактически он создает объект, устанавливает его как контекст во время вызова конструктора (в данном случае `Company`) и
возвращает созданный объект. Именно поэтому сам конструктор ничего не возвращает (хотя может, но это другой разговор), а
внутри константы company оказывается нужный нам объект.

Но не все функции могут быть конструкторами. Отсутствие своего контекста делает невозможным использование оператора new
вместе со стрелочными функциями:

```ts
const f = () => {
}
// TypeError: function is not a constructor
const obj = new f()
```
