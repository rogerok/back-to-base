# Теория: Введение

JavaScript — язык, созданный для выполнения на клиентской стороне. Это сильно повлияло не только на устройство самого языка, но и на то, как устроены его среды исполнения: браузеры, Node.js и другие.

Браузер заточен под **событийную модель**, когда код не исполняется непрерывно, а ожидает пользовательских событий: нажатий на кнопки, кликов мышкой или скроллинга.

Каждое из этих (и десятка других) событий запускает соответствующий обработчик. При этом к обработчику предъявляются важные требования:

- Он **не должен блокировать выполнение**
- Пользователь должен сохранять возможность взаимодействия с интерфейсом
- Несколько задач могут выполняться практически параллельно

---

## Почему JavaScript асинхронный

По этим причинам все движки JavaScript построены по **асинхронной модели**.

Любые операции ввода/вывода (например, выполнение AJAX-запросов) выполняются асинхронно.

Асинхронный код:

- устроен иначе, чем синхронный
- требует другого подхода к обработке ошибок
- меняет архитектурные решения
- требует времени на привыкание

Добавляя одни возможности, он усложняет другие вещи. То, что просто реализуется в синхронном коде, в асинхронном может стать сложнее.

Это цена за производительность и отзывчивость интерфейса.

## Эволюция асинхронности в JavaScript

Асинхронное программирование прошло несколько стадий развития:

1. **Callbacks (Колбеки)**
2. **Promises (Промисы)**
3. **Async/Await**

Каждый из этих подходов важен и используется в современной разработке.

В рамках курса вы:

- Разберётесь в принципах асинхронной модели
- Научитесь правильно обрабатывать ошибки
- Поймёте, как работает Event Loop
- Освоите современный стиль async/await
- Научитесь писать код, который остаётся читаемым и предсказуемым
