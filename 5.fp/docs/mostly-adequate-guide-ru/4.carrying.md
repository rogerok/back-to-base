# [Глава 04: Каррирование](https://github.com/MostlyAdequate/mostly-adequate-guide-ru/blob/master/ch04-ru.md)

Рассмотрим несколько каррированных ф-ций

```js
const match = curry((what, s) => s.match(what));
const replace = curry((what, replacement, s) => s.replace(what, replacement));
const filter = curry((f, xs) => xs.filter(f));
const map = curry((f, xs) => xs.map(f));
```

При их определении намеренно набор данных с которым мы работаем передан последним аргументом.

```js
match(/r/g, 'hello world'); // [ 'r' ]

const hasLetterR = match(/r/g); // x => x.match(/r/g)
hasLetterR('hello world'); // [ 'r' ]
hasLetterR('just j and s and t etc'); // null

filter(hasLetterR, ['rock and roll', 'smooth jazz']); // ['rock and roll']

const removeStringsWithoutRs = filter(hasLetterR); // xs => xs.filter(x => x.match(/r/g))
removeStringsWithoutRs(['rock and roll', 'smooth jazz', 'drum circle']); // ['rock and roll', 'drum circle']

const noVowels = replace(/[aeiou]/ig); // (r,x) => x.replace(/[aeiou]/ig, r)
const censored = noVowels('*'); // x => x.replace(/[aeiou]/ig, '*')
censored('Chocolate Rain'); // 'Ch*c*l*t* R**n'
```

# Применение

С помощью каррирования мы можем создавать новые ф-ции по мере необходимости, просто предоставляя исходным функциям не
все аргументы, как сделали это с `hasLetterR`, `removeStringsWithoutRs` и `censored`.

Этим же приемом, мы можем превратить любую ф-цию, применяемую к единственному элементу, в ту, которая будет применяться
к множеству таких элементов.

```js
const getChildren = x => x.childNodes;
const allTheChildren = map(getChildren)
```

Использование ф-ций с меньшим кол-вом аргументов, чем она ожидает, называется **частичным применением**

Частичная ф-ция позволяет нам избавиться от бойлерплейта. Пример функции выше без каррирования.

```js
const allTheChildren = elements => map(elements, getChildren);
```

Когда мы говорили о чистых функциях, мы говорили, что они принимают только один аргумент и возвращают одно значение.
Каррирование обеспечивает именно такое поведение: с каждым аргументом возвращается новая ф-ция, принимающая ооставшиеся
аргументы.

Функция считается чистой, если для одинаковых аргументов она возвращает одинаковый результат. И не принципиально,
является ли он функцией. Подразумевается, что 2 возвращаемые функции одинаковые, если они производят одинаковый
результат, несмотря на то, что в результате нескольких вызовов будет возвращена не одна и та же функция. То же
справедливо и для других значений-объектов.

Также ради краткости мы допускаем, что каррированная функция может принять несколько аргументов сразу — чтобы в таком
случае не пришлось несколько раз писать `()` (имеется в виду, что каррированная функция, строго говоря, должна принимать
аргументы только по одному)
