## 1. Что такое `IO<A>`

Представь, что ты пишешь **сценарий фильма**, а не снимаешь сам фильм.  
`IO<A>` — это именно сценарий: описание того, что должно произойти.

```
"Напечатай 'Привет'"  →  это IO
"Прочитай строку"     →  это IO
"Верни число 42"      →  это IO
```

Реально ничего не происходит, пока кто-то не запустит интерпретатор (`run`).

---

### Три типа узлов

```
IO<A> — это одно из трёх:

┌─────────────────────────────────────────────────────┐
│  Pure(value)                                        │
│  "Я закончил. Вот результат: value"                 │
│                                                     │
│  ReadLine(next: string => IO<A>)                    │
│  "Прочитай строку от пользователя, затем продолжи"  │
│                                                     │
│  WriteLine(text, next: IO<A>)                       │
│  "Напечатай text, затем продолжи"                   │
└─────────────────────────────────────────────────────┘
```

---

### Дерево программы

Программа "Спроси имя → поздоровайся" выглядит как дерево:

```
WriteLine("What is your name?")
  │
  └─► ReadLine
        │
        └─► (name) ──► WriteLine("Hello, {name}!")
                          │
                          └─► Pure(undefined)   ← конец
```

Это именно то, что записано в `greeting` в коде.

---

## 2. Зачем нужен `bind`

Допустим, у тебя есть две программы:

```
Программа A: IO<string>       Программа B: string => IO<number>
"Прочитай имя от              "Возьми имя, верни его длину
  пользователя"                 как IO<number>"
```

Ты хочешь **склеить их в одну программу** `IO<number>`:  
"Прочитай имя → верни его длину".

Именно это делает `bind`:

```
bind(A, B) = IO<number>

"Сначала выполни программу A, 
 возьми её результат,
 передай в B,
 выполни B"
```

---

### Без bind — руками, это неудобно

```ts
// Приходится строить дерево вручную каждый раз
const program: IO<number> = {
  tag: "readLine",
  next: (name) => ({
    tag: "pure",
    value: name.length
  })
};
```

### С bind — чисто и компонуемо

```ts
const readNameLength: IO<number> = bind(readLine, (name) => pure(name.length));
```

---

## 3. Как bind работает — простая идея

**Идея одна:** найти все `Pure`-листья в дереве и заменить их на `f(value)`.

```
До bind:                      После bind(io, f):

WriteLine("Q?")               WriteLine("Q?")
  └─ ReadLine                   └─ ReadLine
       └─ Pure("Вася")               └─ f("Вася")   ← Pure заменён на f(value)
```

`bind` рекурсивно обходит дерево сверху вниз, пока не дойдёт до `Pure`.

## 4. Разбор каждого case

```ts
export const bind = <A, B>(io: IO<A>, f: (a: A) => IO<B>): IO<B> => {
  switch (io.tag) {
    case "pure":     // ...
    case "readLine": // ...
    case "writeLine":// ...
  }
};
```

---

### `case "pure"` — база рекурсии

```ts
case "pure":
  return f(io.value);
```

```
Нашли Pure(value) — это конец предыдущей программы.
Значение готово. Передаём его в f и получаем следующую программу.

Pure("Вася") + f  →  f("Вася")
```

```
До:            После:

Pure("Вася")   IO<B>   ← то, что вернула f("Вася")
```

---

### `case "writeLine"` — пройти мимо

```ts
case "writeLine":
  return { tag: "writeLine", text: io.text, next: bind(io.next, f) };
```

```
WriteLIne — промежуточный узел. Нам нужно сохранить его,
но спуститься глубже — к next.

WriteLine("Q?", next: Pure("x")) + f
  →  WriteLine("Q?", next: bind(Pure("x"), f))
  →  WriteLine("Q?", next: f("x"))
```

Визуально:

```
До:                         После:

WriteLine("Q?")             WriteLine("Q?")
  └─ Pure("x")                └─ f("x")   ← bind добрался до Pure
```

---

### `case "readLine"` — обернуть продолжение

```ts
case "readLine":
  return { ...readLine, next: (x) => bind(io.next(x), f) };
```

Это самый хитрый case. Почему нельзя написать `bind(io.next, f)`?

```
io.next — это функция (string => IO<A>), а не IO<A>.
Мы не знаем x прямо сейчас — его введёт пользователь в будущем.
```

Поэтому мы оборачиваем:

```
"Когда придёт x — сначала выполни io.next(x), потом примени f"
```

```
До:                                  После:

ReadLine(next: x => Pure(x))         ReadLine(next: x => bind(Pure(x), f))
                                                        = x => f(x)
```

## 5. Полный трейс шаг за шагом

Возьмём программу:

```ts
const program = bind(greeting, () => pure(42));
```

`greeting`:

```
WriteLine("What is your name?")
  └─ ReadLine
       └─ (name) => WriteLine("Hello, {name}!")
                      └─ Pure(undefined)
```

Хотим: после `greeting` вернуть `42`.

---

**Шаг 1** — `bind` встречает `WriteLine`

```
bind(
  WriteLine("What is your name?", next: ReadLine(...)),
  () => pure(42)
)

→ case "writeLine"
→ сохраняем текст, уходим в next

= WriteLine("What is your name?", next: bind(ReadLine(...), f))
```

---

**Шаг 2** — `bind` встречает `ReadLine`

```
bind(
  ReadLine(next: name => WriteLine("Hello, {name}!", next: Pure(undefined))),
  () => pure(42)
)

→ case "readLine"
→ оборачиваем: "когда придёт x — вызови bind на результате"

= ReadLine(next: (x) => bind(
    WriteLine("Hello, {x}!", next: Pure(undefined)),
    () => pure(42)
  ))
```

---

**Шаг 3** — отложен до ввода `x`. Когда пользователь введёт имя:

```
bind(
  WriteLine("Hello, {x}!", next: Pure(undefined)),
  () => pure(42)
)

→ case "writeLine"
→ сохраняем текст, уходим в next

= WriteLine("Hello, {x}!", next: bind(Pure(undefined), () => pure(42)))
```

---

**Шаг 4** — `bind` встречает `Pure`

```
bind(Pure(undefined), () => pure(42))

→ case "pure"
→ f(undefined) = pure(42)

= Pure(42)
```

---

**Итоговое дерево:**

```
WriteLine("What is your name?")
  └─ ReadLine(x =>
       WriteLine("Hello, {x}!")
         └─ Pure(42)            ← вот где пристыковалось
     )
```

`Pure(undefined)` был заменён на `Pure(42)`. `bind` сработал как ожидалось.

## 6. Аналогия из жизни

### `bind` — как оглавление книги с продолжением

Представь, что `IO` — это **квест в текстовой игре**:

```
Страница 1: "Ты входишь в таверну. Напиши своё имя."  → ReadLine
Страница 2: "Привет, {имя}! Иди на страницу 3."       → WriteLine
Страница 3: "Конец."                                   → Pure
```

`bind` — это операция **"вклеить новые страницы после страницы 3"**:

```
Было:  страница 3 = "Конец."
Стало: страница 3 = "Продолжи квест..." → страница 4 → ... → новый "Конец."
```

`bind` обходит всю книгу, находит последнюю страницу и вклеивает продолжение.  
При этом `ReadLine`-страницы особенные: продолжение вклеивается в **каждую ветку** ответа.

---

### `Pure` — это не `return` в обычном смысле

Начинающие путают `Pure` с `return` из JS-функции.

```
// Обычная функция — return сразу передаёт управление
function getName(): string {
  return "Вася";  // выполнилось, вернули значение
}

// IO — Pure это просто тег "я готов"
const getName: IO<string> = { tag: "pure", value: "Вася" };
// Ничего не выполнилось. Это просто структура данных.
```

`Pure` — это **метка на листе дерева**: "здесь значение готово, сюда можно пристыковать продолжение".

## 7. Частые вопросы

**Q: Почему `bind` не запускает эффекты?**

`bind` только строит дерево. Эффекты (реальный вывод в консоль, реальный ввод) запускает `run` — интерпретатор, который обходит готовое дерево.

```
bind  →  строит дерево описания
run   →  выполняет описание
```

---

**Q: Почему в `readLine` case нельзя написать `bind(io.next, f)`?**

`io.next` — это `(string) => IO<A>`, функция, а не `IO<A>`.  
`bind` принимает `IO<A>`, а не функцию. Нам нужно подождать, пока придёт `x`, и только тогда вызвать `io.next(x)`.

```ts
// Неправильно — типы не совпадут
next: bind(io.next, f) // io.next это функция, не IO

// Правильно — оборачиваем в лямбду
next: (x) => bind(io.next(x), f)
```

---

**Q: Чем `bind` отличается от обычного `flatMap` в массивах?**

Это одна и та же идея:

```ts
// Array flatMap
[1, 2, 3].flatMap(x => [x, x * 2])
// = [1, 2,  2, 4,  3, 6]

// IO bind
bind(readLine, name => writeLine(`Hello, ${name}`))
// = ReadLine -> WriteLine -> Pure
```

`flatMap` разворачивает вложенные массивы, `bind` разворачивает вложенные `IO`.  
В обоих случаях: "возьми значение из контейнера, передай в f, f возвращает такой же контейнер — склей их".

---

**Q: Что значит `{ ...readLine, next: ... }`?**

`readLine` — это константа `{ tag: "readLine", next: pure }`.  
Спред `...readLine` копирует `tag: "readLine"`, а потом мы перезаписываем `next`.  
Это просто короткая запись вместо `{ tag: "readLine", next: ... }`.

---

## Итог

| Узел | Что делает bind |
|------|----------------|
| `Pure(value)` | **База:** заменяет `Pure` на `f(value)` |
| `WriteLine(text, next)` | **Проходит:** сохраняет текст, рекурсивно идёт в `next` |
| `ReadLine(next)` | **Откладывает:** оборачивает `next` так, что `f` применится позже |

Ключевая идея — **`bind` не выполняет программу, а перестраивает дерево**, приклеивая продолжение к каждому `Pure`-листу.
