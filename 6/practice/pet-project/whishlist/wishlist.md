# Wishlist Service

## Описание

Сервис для создания списков желаний. Можно создавать несколько вишлистов, добавлять товары со ссылками и ценами, делиться с друзьями. Друзья могут "резервировать" подарки чтобы не было дублей.

---

## Экраны

### 1. Лендинг / приветственная страница (неавторизован)

- Описание сервиса
- Кнопки: войти, зарегистрироваться
- Пример вишлиста (статичный демо)

### 2. Регистрация / Вход

- Форма регистрации: имя, email, пароль
- Форма входа: email, пароль
- Валидация полей в реальном времени

### 3. Мои вишлисты (главная для авторизованного)

- Список своих вишлистов: название, количество позиций, дата события (если указана), статус (публичный/приватный)
- Кнопка "Создать вишлист"
- Кнопка "Поделиться" рядом с каждым списком (копирует ссылку)

### 4. Создание / редактирование вишлиста

- Поле: название (напр. "День рождения 2025")
- Поле: описание (опционально)
- Поле: дата события (опционально)
- Переключатель: публичный / приватный
- Кнопки: сохранить, отмена

### 5. Мой вишлист — детальный просмотр (владелец)

- Название, описание, дата события
- Ссылка для шаринга + кнопка "Копировать"
- Список позиций: название, фото (если есть), цена, ссылка, приоритет, статус резерва (зарезервировано / нет)
- Кнопка "Добавить позицию"
- Редактирование и удаление каждой позиции
- Сортировка по приоритету / цене

### 6. Добавление / редактирование позиции

- Поле: название товара
- Поле: ссылка на товар (URL)
- Поле: цена (опционально)
- Поле: приоритет (низкий / средний / высокий)
- Поле: комментарий (опционально)
- Загрузка фото или автоматическое получение превью по URL (og:image)

### 7. Публичный вишлист (вид гостя / друга)

- Название вишлиста, имя владельца, дата события
- Список позиций: название, фото, цена, ссылка, приоритет
- Кнопка "Зарезервировать" на каждой позиции
  - Позиция помечается как зарезервированная (без раскрытия кто зарезервировал — для сюрприза)
  - Гость видит "уже зарезервировано" на чужих резервах
  - Гость может отменить свой резерв
- Фильтр: показать только свободные

### 8. Профиль пользователя

- Имя, email
- Смена пароля
- Удаление аккаунта

## Функционал

### Аутентификация
- Регистрация / вход по email + пароль
- JWT-токены, refresh механизм
- Защищённые роуты

### Вишлисты
- Несколько вишлистов на пользователя
- Публичные (доступны по ссылке без логина) и приватные
- Уникальная короткая ссылка на каждый вишлист

### Позиции вишлиста
- CRUD позиций
- Приоритет (влияет на сортировку)
- Ссылка на товар с автопревью (og:image / og:title парсинг по URL)
- Ручная загрузка фото

### Резервирование
- Любой человек по ссылке может зарезервировать позицию
- Владелец НЕ видит кто зарезервировал (сюрприз)
- Владелец видит только факт резерва
- Гость видит свои резервы и может их отменить

### Уведомления (опционально, v2)
- Email за N дней до даты события
- Напоминание что список не обновлялся давно

## Технические особенности

- Парсинг og:image и og:title по ссылке на бэке (Fastify)
- Короткие уникальные ID для публичных ссылок (nanoid)
- Валидация через zod
- Загрузка изображений (multipart/form-data через @fastify/multipart)
